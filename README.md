# AI PR Reviewer Action

An LLM-backed GitHub Action that reviews pull requests using project context (`project.md`), PR metadata, changed file diffs, and scan logs, then posts actionable feedback.

## Features
- Pulls `project.md` from the base branch to respect architecture/constraints.
- Reads PR title/body, changed files (with truncated patches), and optional scan logs.
- Builds a structured prompt and calls an OpenAI-compatible chat endpoint.
- Posts a PR comment (or job summary) with verdict, key issues, and test recommendations.

## Inputs
- `llm-endpoint` **(required)**: Base URL to a chat completion endpoint (OpenAI compatible).
- `llm-api-key` **(required)**: API key for the endpoint.
- `model` (default `gpt-4o-mini`): Model name.
- `project-md-path` (default `project.md`): Path to the architecture doc.
- `scan-log-paths` (optional): Comma-separated file paths to logs (e.g., `reports/sast.txt,reports/tests.txt`).
- `max-files` (default `50`): Maximum files to review.
- `max-hunk-size` (default `8000`): Patch characters kept per file.
- `post-mode` (default `comment`): `comment` posts to PR; `summary` writes to job summary.

## Example workflow
```yaml
name: AI PR Reviewer

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/checkout@v4

      - name: AI Assisted PR Reviewer
        uses: SarvariHarshitha/ai-code-review@1.0.0   
        with:
          llm-endpoint: ${{ secrets.LLM_ENDPOINT }}          # OpenAI-compatible chat endpoint
          llm-api-key: ${{ secrets.LLM_API_KEY }}            # API key for the endpoint
          model: gpt-4o-mini                                 # or deepseek-chat, etc.
          project-md-path: project.md
          scan-log-paths: reports/sast.txt,reports/tests.txt # optional
          post-mode: comment                                 # or summary
          max-files: 50
          max-hunk-size: 8000
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Development
```bash
npm install
npm run build
```
- Entry point: `src/index.ts`
- Output: `dist/index.js` (generated by `npm run build`)
- Runtime: node20

## Notes
- Ensure `project.md` exists on the base branch; otherwise the action notes it as missing.
- Trim or summarize scan logs to keep prompts small.
- Use org secrets for `LLM_ENDPOINT` and `LLM_API_KEY`.
- Demo: this repo includes `reports/sast.txt` and `reports/tests.txt` plus this README edit to exercise the AI reviewer.
- DeepSeek: set `llm-endpoint` to `https://api.deepseek.com/chat/completions` (the action will also append `/v1/chat/completions` if you pass the base URL only) and `model` to `deepseek-chat`.
